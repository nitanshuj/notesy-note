# Commands for Running the Application

This document details the common commands required to set up, run, and build the Notesy Note Tauri application.

## Prerequisites

Before running the application, make sure you have the following installed:
- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- [Rust](https://www.rust-lang.org/) (via `rustup`)
- Build tools for your operating system:
  - **Windows**: [Build Tools for Visual Studio](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (make sure C++ build tools are selected)
  - **macOS**: Xcode Command Line Tools (`xcode-select --install`)
  - **Linux**: System packages like `build-essential`, `curl`, `wget`, `libssl-dev`, `libgtk-3-dev`, `webkit2gtk-4.0` (or `webkit2gtk-4.1` depending on distro)

---

## 1. Setup & Installation

To install all Node.js dependencies:

```bash
npm install
```

---

## 2. Development

### Run the Desktop Application (Tauri + React)
To start the application in development mode with hot-reloading for both Rust and the React frontend:

```bash
npm run tauri dev
```
*(Alternatively: `npx tauri dev`)*

### Run Web-Only Frontend (React)
If you only need to work on the UI styling/layout and do not require Tauri native APIs (e.g. filesystem access, dialogs):

```bash
npm run dev
```

---

## 3. Production Build

To compile and package the application into a production-ready installer/executable:

```bash
npm run tauri build
```
*(Alternatively: `npx tauri build`)*

The output bundle (installer/executable) will be generated under the `src-tauri/target/release/bundle/` directory.

---

## 4. Other Utilities

### Preview Web Production Build
To build the frontend and preview it locally in a browser:

```bash
npm run build
npm run preview
```

### Tauri Info / Environment Diagnostics
To check the version of Tauri, Rust, and other system dependencies for debugging:

```bash
npx tauri info
```

---

## 5. Troubleshooting

### Stale Build Cache / Directory Rename Error
If you rename or move the project directory, you may encounter an error like:
```text
failed to read plugin permissions: failed to read file '...\target\debug\build\tauri-...\out\permissions\app\autogenerated\commands\app_hide.toml': The system cannot find the path specified (os error 3)
```
This happens because Cargo cached absolute paths in the `target` build directory pointing to the old directory location.

**Solution:**
Clean the cargo build artifacts to force a fresh build. Run the following command from the root directory:

```bash
# Clean Tauri/Rust build artifacts
npx tauri clean
```
*(Alternatively, you can delete the `src-tauri/target` directory manually or run `cargo clean` inside `src-tauri`)*
